This is a fantastic use case, and yes — Channels is basically the missing piece that makes this possible. Here's how the whole system would work.

The architecture at a high level:

Your home machine runs Claude Code with --channels and the Telegram plugin, keeping a session alive. Your phone talks to that session via a Telegram bot. The missing glue is voice-in and voice-out on the phone side.

The practical stack:

Claude Code + Telegram Channel (home machine): Set up the Telegram bot via BotFather, install the plugin, and run claude --channels plugin:telegram@claude-plugins-official in a persistent terminal (tmux/screen so it survives). This gives you a two-way text bridge — anything you send the bot goes into the live session, and Claude's reply comes back through Telegram.
Voice layer on your phone: This is where it gets interesting. You have a few options depending on your phone OS: iOS path — Siri Shortcuts is probably the fastest to prototype. You can build a Shortcut that does: "Dictate Text" → "Send Message via Telegram" (or hit the Telegram Bot API directly via HTTP). For the response, Telegram notifications would arrive, and you could build a second shortcut or use the "Speak Text" action. The friction here is that Shortcuts doesn't natively listen for Telegram responses in a loop. Android path — Tasker + AutoVoice gives you way more control. Tasker can intercept Telegram notifications, extract the text, and pipe it through TTS. It can also map Bluetooth button events to start/stop voice recording and send the transcription to the bot API. Custom lightweight app is the cleanest long-term solution. A simple React Native app (or even a PWA) that does: hold button → record → on release, hit a speech-to-text API (Whisper, Deepgram, or just the device's native speech recognition) → send transcribed text to Telegram Bot HTTP API → listen for responses → TTS playback. This is probably 200-300 lines of actual code.
Bluetooth remote on the handlebars: Most cheap Bluetooth camera remotes or media remotes send standard HID events (like media play/pause or volume keys). On Android, Tasker can intercept these and trigger any action. On iOS, it's more limited — a Bluetooth shutter button triggers the camera, but you could use an AirPods squeeze or a dedicated BLE button that talks to your custom app.
If you want to build the custom app route, the core loop is:

BT button press → start recording audio
BT button press again → stop recording
  → run speech-to-text (on-device or API)
  → POST to https://api.telegram.org/bot<TOKEN>/sendMessage 
    with chat_id and transcribed text
  → poll or webhook for bot responses
  → pipe response text through TTS → audio out to headphones
What I'd recommend as the quickest path to a working prototype: Start with the Telegram channel running on your machine, then use Tasker (if Android) or Shortcuts (if iOS) to wire up voice dictation → send to bot. That gets you 80% of the way there without writing any app code. Once you validate the loop feels good while riding, then build the proper app with BLE remote support.
